DNS setup & verify

Copy each record into the Porkbun dashboard, add it manually, then click Verify DNS to check it went live. Verification uses public DNS-over-HTTPS (Google) — no backend, works offline-of-our-servers.

Order: add SPF + DMARC first, monitor ~2 weeks, then raise DMARC to p=reject. Source: AEO audit DNS Analyzer (28/100).

How to apply

  1. Create the dmarc-reports@ alias in ForwardEmail first (the DMARC rua mailbox must exist).
  2. Add the two critical records (root SPF, DMARC), then click Verify DNS.
  3. Wait for reports to look clean (~2 weeks), then change DMARC to p=reject.
  4. Add the harden records. Add CAA only after confirming your host's cert CA.
