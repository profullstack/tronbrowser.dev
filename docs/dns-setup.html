DNS records to add — tronbrowser.dev

Add these in the Porkbun DNS dashboard. Copy buttons grab the exact record value.

Source: AEO audit DNS Analyzer (28/100 — no SPF, DMARC p=none). Order: add SPF + DMARC first, monitor two weeks, then move DMARC to p=reject.

Root SPF critical

Stops anyone from spoofing @tronbrowser.dev. Add any senders you actually use before -all (e.g. include:amazonses.com for SES, include:_spf.resend.com for Resend).

SPF (TXT on root)
Host@
TypeTXT
Valuev=spf1 include:forwardemail.net -all

DMARC policy critical

Upgrades from p=none (no enforcement) to quarantine, with aggregate reporting. The rua mailbox must exist — set up a catch-all or alias in ForwardEmail first.

DMARC (TXT)
Host_dmarc
TypeTXT
Valuev=DMARC1; p=quarantine; rua=mailto:dmarc-reports@tronbrowser.dev; aspf=r; adkim=r;

send subdomain SPF harden

The send subdomain currently uses ~all (softfail). Tighten to -all if it only sends via SES.

send SPF (TXT)
Hostsend
TypeTXT
Valuev=spf1 include:amazonses.com -all

CAA harden

Restricts which CAs may issue certs.

⚠ Verify your host's CA first. Railway/its edge issues the TLS cert for tronbrowser.dev — if it does not use Let's Encrypt, this record will BREAK cert renewal. Confirm before adding.

CAA — issue
Host@
TypeCAA
Value0 issue "letsencrypt.org"
CAA — issuewild (disallow wildcards)
Host@
TypeCAA
Value0 issuewild ";"

TLS-RPT harden

Reports on inbound TLS delivery failures. Mailbox must exist.

TLS-RPT (TXT)
Host_smtp._tls
TypeTXT
Valuev=TLSRPTv1; rua=mailto:tls-reports@tronbrowser.dev

MTA-STS harden

Enforces TLS for inbound mail. Also needs an HTTPS policy file served at https://mta-sts.tronbrowser.dev/.well-known/mta-sts.txt. Bump the id whenever the policy changes.

MTA-STS (TXT)
Host_mta-sts
TypeTXT
Valuev=STSv1; id=20260701000000

ForwardEmail DKIM harden

Copy the selector name and public key from the ForwardEmail dashboard (app.forwardemail.net), then publish:

ForwardEmail DKIM (TXT)
Host<selector>._domainkey
TypeTXT
Valuev=DKIM1; k=rsa; p=<key from dashboard>

How to apply

  1. Add the two critical records (root SPF, DMARC). Create the dmarc-reports@ alias in ForwardEmail first.
  2. Wait ~48h, confirm mail still delivers and check the DMARC aggregate reports.
  3. Add the harden records (send SPF, TLS-RPT, MTA-STS, DKIM). Add CAA only after confirming your cert CA.
  4. After ~2 weeks of clean reports, change DMARC to p=reject.
  5. Re-run the DNS Analyzer to confirm the score climbs from 28/100.
